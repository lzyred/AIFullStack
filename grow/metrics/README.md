# AI Product Metrics Framework

> Andrew Chen test: concrete definitions and numbers — not platitudes like "focus on user experience."

---

## The Growth Accounting Framework

The foundation for understanding your metrics: what actually drives MAU change.

```
Net MAU change = (new users + reactivated users) - churned users

Growing  = positive terms > negative terms
Stable   = positive terms ≈ negative terms
Shrinking = positive terms < negative terms
```

**The critical insight**: many products look stable on MAU while actually running "massive churn + massive new-user backfill." This is unsustainable — you're running on a treadmill, not moving forward.

## Core Metric Definitions

### 1. Activation Rate

**Definition**: the percentage of registered users who complete the "core action."

The core action must directly reflect product value — not account creation:

```python
# Wrong activation definition:
activation = "completed email verification"  # that's registration, not activation

# Correct activation definitions (examples):
activation = {
    "AI writing tool": "completed first AI generation with output > 100 words",
    "RAG Q&A system": "asked a question and received an answer with document citations",
    "AI coding assistant": "accepted the first AI code suggestion",
}
```

**Alert threshold**: activation rate < 20% — fix onboarding immediately.

### 2. Retention Rate (D1 / D7 / D30)

**Definition**: the percentage of registered users who are still active on day N (calculated from each user's registration date, not a rolling window).

```python
def calculate_retention(users_df, events_df, day: int) -> float:
    """
    users_df: user_id, registration_date
    events_df: user_id, event_date
    """
    target_date = users_df['registration_date'] + pd.Timedelta(days=day)

    active_on_day = events_df.merge(
        users_df[['user_id', 'registration_date']],
        on='user_id'
    )
    active_on_day = active_on_day[
        pd.to_datetime(active_on_day['event_date']).dt.date ==
        (pd.to_datetime(active_on_day['registration_date']) + pd.Timedelta(days=day)).dt.date
    ]['user_id'].nunique()

    return active_on_day / len(users_df)
```

**Alert threshold**: D30 < 5% means no PMF — don't scale acquisition.

### 3. MAU / Registered User Ratio

**Definition**: users active in the last 30 days ÷ total registered users.

**Thresholds**: < 10% is a serious warning; > 30% indicates a healthy product; > 50% is Facebook-level.

### 4. Viral Coefficient k

**Definition**: the average number of new registered users generated by each existing user.

```python
k = invites_sent_per_user * invite_conversion_rate

# k > 1: super-growth (each user brings in more than 1 new user)
# k = 0.5: healthy viral supplement (every 2 users bring 1 new user)
# k < 0.1: almost no viral spread — entirely dependent on paid acquisition
```

### 5. LTV / CAC by Channel

**Definitions:**
- **LTV** (lifetime value) = average monthly ARPU ÷ monthly churn rate (more accurate formula)
  - When early data is limited, use a conservative estimate: average monthly ARPU × estimated months of use (extrapolated from known retention data, take the conservative end)
  - Note: simply using "ARPU × average retention duration" overstates LTV because it ignores discount rates — especially problematic for B2B products with long revenue cycles
- **CAC** (customer acquisition cost) = total channel spend ÷ **activated** users from that channel (not registrations)

**Alert threshold**: channels with LTV/CAC < 3 are generally not sustainable. Always use conservative LTV — an LTV/CAC > 3 built on optimistic assumptions is meaningless.

## PMF Signal Criteria

### Quantitative Signals (all must be met)

- D30 retention ≥ 15% **and** the retention curve has flattened (stopped declining)
- Among organically retained users, > 40% use the product ≥ 3 times per week
- Organic (unpaid) sources account for > 30% of new users

### Qualitative Signals

- Customers ask about pricing in sales conversations — you're not chasing them
- Users start requesting features "not on the roadmap" (usage depth has exceeded expectations)
- Spontaneous word-of-mouth appears (users recommend to colleagues without being asked)
- Sean Ellis test (supporting signal, not a hard gate): ask "How would you feel if this product disappeared tomorrow?" — > 40% answering "very disappointed" is a PMF signal, but B2B tool products may score lower due to survey design or user composition; combine with quantitative signals

## Metric Priorities by Growth Stage

```
Stage 0: 0–100 users
  Focus: qualitative > quantitative
  Core question: are users solving a real problem?
  Track: user interview frequency, successful manual deliveries

Stage 1: 100–10K users
  Focus: retention above everything
  Core metric: D30 retention rate and curve shape
  Warning: don't spend energy on growth yet — lock in retention first

Stage 2: 10K–100K users
  Focus: activation rate + channel quality
  Core metrics: activation rate, LTV/CAC by channel
  Goal: find a repeatable, scalable acquisition loop

Stage 3: 100K+ users
  Focus: growth loop efficiency + monetization
  Core metrics: growth accounting (new vs. churned), k coefficient, power user ratio
  Goal: keep positive terms > negative terms
```

## 6 Common Metric Mistakes

### 1. Using MAU to Hide a Retention Problem

Continuous ad spend can make MAU look healthy, but when the retention curve trends toward zero, MAU collapses the moment you stop buying traffic. **Look at the retention curve before looking at MAU.**

### 2. Treating Registrations as Activations

A Product Hunt launch can deliver 5,000 sign-ups but only 200 people who actually use the product. Track **activated users**, not registrations.

### 3. Relative Improvement vs. Absolute Impact

A feature change lifts conversion 50%, but only affects 0.01% of users → negligible absolute impact. Another change lifts conversion 5% but affects 50% of users → 5x larger absolute impact. **Always measure absolute user count affected, not just relative percentage lift.**

### 4. Ignoring Channel Concentration Risk

80% of traffic from a single source (Google SEO, iOS push, one partner) = extreme risk. Monitor channel share and alert when any single source exceeds 60%.

### 5. Averages Without Cohorts

```python
# Wrong:
avg_d30 = total_active_d30 / total_users  # averages hide trends

# Right: segment by registration date
retention_by_cohort = {
    "2026-01": 0.30,  # older users retain well
    "2026-02": 0.25,
    "2026-03": 0.18,
    "2026-04": 0.12,  # new user retention is deteriorating
    "2026-05": 0.08,  # alarm
}
# Average = 0.19 — looks fine, but the product is actually regressing
```

### 6. Viral Coefficient Masking Low Retention

k > 1 but D30 < 5%: the product is growing fast and burning a hole just as fast. The Viddy case. **k is a multiplier — it amplifies retention and churn equally.**

## Minimum Viable Dashboard

You need this from day one of launch:

```python
dashboard_metrics = {
    # Updated daily
    "daily": [
        "new registrations",
        "new activations (not registrations)",
        "DAU",
        "API call volume and cost",
        "error rate",
    ],

    # Updated weekly
    "weekly": [
        "D7 retention rate (broken down by acquisition channel)",
        "activation rate by channel",
        "user cohort retention curves",
        "power user ratio (5+ sessions/week)",
    ],

    # Updated monthly
    "monthly": [
        "D30 retention rate and trend",
        "MAU / registered user ratio",
        "LTV and CAC estimates",
        "NPS or Sean Ellis test",
    ]
}
```
