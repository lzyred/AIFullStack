## 增长会计框架

理解指标的基础：你的 MAU 变化等于什么。

```
净 MAU 变化 = (新用户 + 重激活用户) - 流失用户

增长 = 正向项 > 负向项
平稳 = 正向项 ≈ 负向项
衰退 = 正向项 < 负向项
```

**关键洞见**：很多产品 MAU 表面平稳，实际上是"大量用户流失 + 大量新用户填补"，这是不可持续的——你在跑步机上跑，不是在向前走。

## 核心指标定义

### 1. 激活率

**定义**：注册后完成"核心行为"的用户比例。

核心行为必须是产品价值的直接体现，不是注册账号：

```python
# 错误的激活定义：
activation = "完成邮箱验证"  # 这是注册，不是激活

# 正确的激活定义（举例）：
activation = {
    "AI 写作工具": "完成第一次 AI 生成，且生成结果超过 100 字",
    "RAG 问答系统": "提问并收到包含文档引用的回答",
    "AI 代码助手": "接受了第一个 AI 代码建议",
}
```

**警戒值**：激活率 < 20% 需立即优化 Onboarding。

### 2. 留存率（D1/D7/D30）

**定义**：第 N 天仍有登录/使用行为的注册用户比例（按注册日期计算，不是滚动 N 天）。

```python
def calculate_retention(users_df, events_df, day: int) -> float:
    """
    users_df：user_id, registration_date
    events_df：user_id, event_date
    """
    target_date = users_df['registration_date'] + pd.Timedelta(days=day)

    active_on_day = events_df.merge(
        users_df[['user_id', 'registration_date']],
        on='user_id'
    )
    active_on_day = active_on_day[
        pd.to_datetime(active_on_day['event_date']).dt.date ==
        (pd.to_datetime(active_on_day['registration_date']) + pd.Timedelta(days=day)).dt.date
    ]['user_id'].nunique()

    return active_on_day / len(users_df)
```

**警戒值**：D30 < 5% 说明没有 PMF，不要扩大获客。

### 3. MAU/注册用户比

**定义**：过去 30 天活跃用户 ÷ 总注册用户数。

**警戒值**：< 10% 严重警告；> 30% 说明产品健康；> 50% 是 Facebook 级别。

### 4. 病毒系数 k

**定义**：每个用户平均带来的新注册用户数。

```python
k = 邀请数量 × 邀请转化率

# k > 1：超级增长（每个用户带来超过 1 个新用户）
# k = 0.5：健康的病毒补充（每 2 个用户带来 1 个新用户）
# k < 0.1：几乎无病毒传播，完全依赖付费获客
```

### 5. 渠道 LTV/CAC

**定义**：
- LTV（用户生命周期价值）= 平均月 ARPU ÷ 月流失率（更准确的算法）
  - 早期数据不足时保守估算：平均月 ARPU × 预估使用月数（用已知留存数据外推，取保守值）
  - 注意：简单用"ARPU × 平均留存时长"会高估，因为未计入折现率，特别是 B2B 长周期产品
- CAC（获客成本）= 渠道总花费 ÷ 该渠道带来的**激活**用户数（不是注册用户）

**警戒值**：LTV/CAC < 3 的渠道通常不可持续。LTV 必须用保守数字，用乐观 LTV 算出的 LTV/CAC > 3 没有意义。

## PMF 判断标准

### 定量信号（需要全部满足）

- D30 留存 ≥ 15%，**且**留存曲线趋于平稳（停止下滑）
- 自然留存用户中 > 40% 每周使用 ≥ 3 次
- 有机增长（非付费）占新用户 > 30%

### 定性信号

- 客户在销售对话中主动问价格，而不是你追着问
- 用户开始要求你做"不在计划中"的功能（说明使用深度超出预期）
- 出现自发口碑推荐（用户未被邀请就推荐给同事）
- Sean Ellis 测试（辅助参考，不作为硬性门槛）：问"如果明天这个产品消失了，你有多失望？" — > 40% 回答"非常失望"是 PMF 信号，但 B2B 工具类产品可能因问卷设计或用户构成而偏低，需结合定量指标综合判断

## 增长阶段的指标优先级

```
阶段0：0-100 用户
  重点：定性 > 定量
  核心问题：用户是否在解决一个真实问题？
  追踪：用户访谈频率，手动成功交付次数

阶段1：100-10K 用户
  重点：留存 > 一切
  核心指标：D30 留存率和曲线形状
  警告：此阶段不要花精力做增长，先固定留存

阶段2：10K-100K 用户
  重点：激活率 + 渠道质量
  核心指标：激活率、各渠道 LTV/CAC
  目标：找到可重复、可规模化的获客循环

阶段3：100K+ 用户
  重点：增长循环效率 + 变现
  核心指标：增长会计（新增 vs 流失）、k 系数、高频用户比例
  目标：让正向项 > 负向项
```

## 6 个常见指标误区

### 1. 用 MAU 掩盖留存问题

持续投广告可以让 MAU 好看，但留存曲线趋向 0 时，停止投放 MAU 立刻崩盘。**先看留存曲线，再看 MAU。**

### 2. 把注册量当激活量

Product Hunt 能带来 5000 注册，但只有 200 人真正使用产品。追踪**激活用户数**，不是注册数。

### 3. 相对增长 vs 绝对影响

某功能改动使转化率提升 50%，但该功能只影响 0.01% 用户 → 绝对影响极小。另一功能改动提升 5%，但影响 50% 用户 → 绝对影响 5 倍大。**永远看绝对影响人数，不是相对提升率。**

### 4. 忽略渠道集中度风险

80% 流量来自单一来源（Google SEO / iOS 推送 / 某个合作伙伴）= 极高风险。监控各渠道占比，设置 > 60% 告警。

### 5. 只看平均值，不看队列

```python
# 错误做法：
avg_d30 = total_active_d30 / total_users  # 平均值掩盖趋势

# 正确做法：按注册时间分队列
retention_by_cohort = {
    "2026-01": 0.30,  # 老用户留存好
    "2026-02": 0.25,
    "2026-03": 0.18,
    "2026-04": 0.12,  # 新用户留存恶化！
    "2026-05": 0.08,  # 警报！
}
# 平均值 = 0.19，看起来还行，但实际上产品在退步
```

### 6. 病毒系数 k 掩盖低留存

k > 1 但 D30 < 5%：产品在快速增长，也在快速烧洞。Viddy 的案例。**k 是放大器，放大的是留存，也是流失。**
